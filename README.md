I created a music player app using vanilla HTML CSS and Javascript. Users can pause, play and skip songs, display song lyrics, create edit and delete playlist, search songs by title artist or album, upload their own songs, and navigate using app specific keyboard controls.
All user information is stored on a Supabase database. This includes uploaded songs, playlists, usernames and passwords.  Each song has an ID number, title, artist, album, lyrics, and src. This applies to the test songs that are available to everyone and user uploaded songs. Only the test songs are available to all users. User uploaded content, and user created playlists can only be seen on the account that created them. The functions work through seven separate js files that all connect to index.html which connects to three css files. The files are as follows
app.js — The main UI coordinator. Initialises all other modules, wires up event listeners (navigation, player controls, search, playlist management, the add-song modal), and contains all DOM rendering functions (renderHome, renderPlaylist, renderSearchResults).
  auth.js — Authentication UI and session bootstrap. Handles the sign-in/sign-up tab switching, form submission, logout, and account deletion. Also contains the DOMContentLoaded entry point that checks for an existing session and launches the app if the user is already logged in.
    player.js — Manages audio playback state. Wraps the <audio> element to handle load, play, pause, toggle, and skip. Restores the last-played song and position on init, auto-advances through playlists when a song ends, and saves state to localStorage after every action.
  playlist.js — A thin wrapper around the Supabase playlist functions from storage.js. Exposes clean async methods — getAll, create, removePlaylist, addSong, removeSong, getSongs — so the rest of the app doesn't call storage directly.
  search.js — Real-time song filtering. Listens to the search input and filters the in-memory song list by title, artist, or album on every keystroke, then fires a callback with the matching results.
  lyrics.js — Controls the lyrics side-panel. Handles open/close/toggle, and re-renders the panel content whenever the current song changes — splitting the lyrics string into line-by-line elements and tagging section labels (lines starting with [) with a special CSS class.
storage.js — All data persistence. Contains two layers: Supabase (auth functions, song upload/save/fetch, playlist CRUD) and localStorage (saving and restoring the player's playback state between sessions).
The exposed key in storage.js is not the service_role key but the anon key which does not have security clearances, is recognised as an anonymous/public client, and is documented by Supabase as safe for client-side code. Without this key available the Netlify deployment would not function without major restructuring.  
To run the app all you would need to do is follow the netlify link or download the github repository and run locally. This will allow you to create an account and act as a user but not to edit other user accounts.
